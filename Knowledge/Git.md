## `git rebase` 和 `git merge` 的区别
  1. `git rebase` 和 `git merge` 都是用来把 两个分支的改动合并在一起 的，但它们处理历史的方式不一样。
    1. `git merge` 保留分支历史，产生一个新的合并提交（merge commit）
      1. 好处：历史完整，不会丢信息。
      2. 坏处：分支多了以后，历史会变得 很复杂，像网状图。
    2. `git rebase` 把分支的提交“搬到”另一个分支的最新提交之后，保持线性历史
      1. 好处：历史清晰、直线化，看起来像一个人写的
      2. 坏处：会改写提交历史，如果别人也在用这个分支，容易出问题
  2. 最佳实践
    1. 开发自己功能分支时，可以多用 **rebase**，保持历史整洁。
    2. 合并到主分支（main/master）时，通常用 **merge**，保留历史，不容易出问题。
